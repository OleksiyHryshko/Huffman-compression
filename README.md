# Huffman Compression (C++)

> A lightweight, command-line file compressor/decompressor written in C++ that implements Huffman coding.

This was initially a first-semester programming lab project. I am keeping it public as a learning artifact: the core algorithm works, the codebase is clean, and it demonstrates a solid grasp of fundamental data structures. I have documented the current limitations and a roadmap for future improvements below.

## ⚙️ What it does
Given an input file, the program:
1. Counts byte frequencies.
2. Builds a Huffman tree using a min-priority queue.
3. Generates a prefix-free bit code for each byte.
4. Writes a **compressed binary output file** and a **dictionary file** (mapping byte -> bitstring).

It also supports full decompression using the generated dictionary.

## 🚀 Build & Usage

### Compilation (macOS / Linux)
You can compile the project directly using `clang++` or `g++`:

\`\`\`bash
# macOS (clang++)
clang++ -std=c++17 -O2 -Wall -Wextra main.cpp Header.cpp -o huffman

# Linux (g++)
g++ -std=c++17 -O2 -Wall -Wextra main.cpp Header.cpp -o huffman
\`\`\`

### Command Line Arguments
* `-i` : Input file
* `-o` : Output file
* `-s` : Dictionary file
* `-t` : Mode (`k` for compression, `d` for decompression)

**To Compress:**
\`\`\`bash
./huffman -i input.txt -o compressed.bin -s dict.txt -t k
\`\`\`

**To Decompress:**
\`\`\`bash
./huffman -i compressed.bin -o restored.txt -s dict.txt -t d
\`\`\`

## 🧠 Implementation Notes
* **Tree Building:** The Huffman tree is constructed using a `std::priority_queue` (min-heap via a custom comparator).
* **Code Generation:** Codes are generated by recursive traversal: the left edge appends `0`, the right edge appends `1`.
* **Binary Packing:** The compressed output is written as raw bytes by taking the bitstring and packing it into 8-bit chunks.

## ⚠️ Known Limitations 
This is an early student project, and there are architectural limitations I would address in a production environment:

* **Bit Padding is not stored:** Compression packs bits into bytes in chunks of 8. If the total bit count isn't a multiple of 8, the last chunk is ambiguous during decompression. *Fix: Store the number of valid bits in the last byte (or store the original file size).*
* **Decompression is O(n · m):** Decoding currently scans the entire dictionary for each growing bit prefix (`for code in dictionary...`). *Fix: Build an explicit decoding trie/tree from the codes for linear-time decoding.*
